Age imputation — step by step

1
Extract title from Name
Regex: ([A-Za-z]+)\. → Mr, Mrs, Miss, Master, Dr, Rev…
2
Consolidate rare titles
Countess/Capt/Jonkheer → "Rare". Mlle/Ms → "Miss". Mme → "Mrs"
3
Impute by title median
Master≈5, Miss≈22, Mr≈30, Mrs≈35. Better than global median (28)
4
Create AgeKnown flag
Binary: was age originally present? Preserves missingness signal

Feature engineering

12 raw features → 18 engineered features. Estimated importance scores from Random Forest Gini impurity.

Estimated feature importance

New features to create

1
Title (from Name)
Mr · Mrs · Miss · Master · Rare. Encodes age, gender, and social class simultaneously.
2
FamilySize = SibSp + Parch + 1
Combined family size. Solo (=1) is a strong negative predictor.
3
IsAlone (binary)
1 if FamilySize == 1. Directly captures the solo-traveller disadvantage.
4
AgeBin (5 bands)
Child/Teen/YoungAdult/Middle/Senior. Captures non-linear age effects.
5
FareBin (quartile cut)
Equal-frequency binning handles skew. Q4 (top 25%) → highest survival.
6
HasCabin + Deck (A–G, U)
Cabin presence binary + deck letter both add independent signal.

Encoding & transformations

7
Sex → binary (female=1, male=0)
Already binary — no one-hot needed. Strongest single predictor in dataset.
8
Embarked → one-hot (drop S)
Drop Southampton (mode) to avoid multicollinearity → Embarked_C, Embarked_Q.
9
log1p(Fare)
Compresses extreme right tail — critical for logistic regression and SVM.
10
Pclass × Sex interaction
3rd-class male <15%; 1st-class female >96%. Interaction captures this joint effect.
11
Standard scaling
Scale Age and log(Fare) for gradient-based models. Tree models don't require it.
12
Drop raw high-cardinality cols
Remove Name, raw Ticket, raw Cabin after engineering. PassengerId always dropped.

Model selection

7 candidate models evaluated — ensemble approach recommended for production accuracy.

Why each model works

★
XGBoost — best single model
Gradient boosting handles non-linear interactions, missing values internally, and is robust to outliers. Built-in L1/L2 regularisation prevents overfitting on 891 rows. Top Kaggle Titanic choice.
★
Random Forest — most interpretable tree ensemble
100+ trees average out variance. Feature importance directly from Gini impurity. Good stable baseline. Less prone to overfitting than single decision trees.
★
Voting Ensemble — production recommendation
Soft-voting over XGBoost + RF + LR reduces model-specific variance. 1–2% accuracy gain over best single model. Use this in your final Streamlit app.
→
Logistic Regression — glass-box baseline
Coefficients interpret as log-odds. Shows direction of each feature. Requires scaling. Cannot capture non-linear interactions without manual polynomial terms.
→
SVM (RBF kernel) — strong with small datasets
Maximises margin in feature space. Works well with 891 rows. Needs carefully tuned C and gamma via cross-validation grid search.

Evaluation strategy

5-fold stratified CV AUC-ROC primary metric Accuracy secondary Precision-Recall for class imbalance GridSearchCV / Optuna for tuning SHAP for feature explanations

Deployment plan

Three deployment tiers — from instant free hosting to production-grade containerised services.

Tier 1 — Streamlit Community Cloud (free, 5 min)
Push code to GitHub → connect to share.streamlit.io → auto-deploy. requirements.txt auto-installed. Free subdomain. Best for demo and portfolio. Zero infrastructure cost.
Tier 2 — Docker + Hugging Face Spaces
Containerise: FROM python:3.11-slim → pip install → EXPOSE 8501. Deploy to HuggingFace Spaces free tier or AWS EC2 t2.micro. Better for heavier workloads or persistent model storage.
Tier 3 — FastAPI + React (production)
Separate ML backend (FastAPI /predict endpoint + joblib model) from frontend. API on Railway/Render, frontend on Vercel. Enables mobile clients, rate limiting, auth, and A/B model testing.
Model versioning & monitoring
Track experiments with mlflow. Log prediction distributions with evidently-ai to detect data drift. Trigger retraining when AUC drops below 0.82 threshold.
Deployment checklist
requirements.txt pinned Model saved to /models Preprocessing pipeline serialised st.cache_data on load functions Error handling on predict GitHub Actions CI on push README with usage instructions
